Anomaly Detection — Design

lib/anomaly-detector.js — subscribes to store, fires alerts

How it works — pipeline
1. MetricsStore emits sample event
Every pollInterval (2s default) — AnomalyDetector receives full snapshot
↓
2. Per-metric evaluators run
🔥 CPU spike — rolling 60s avg > threshold OR > μ + Nσ 📈 Mem leak — linear regression on heap samples, slope > limit ⏱ Loop block — sustained lag > threshold for duration window 📊 Traffic anomaly — msg/s drops or spikes vs rolling baseline
↓
3. Anomaly confirmed → cooldown check
60s cooldown per metric — prevents alert storm. One anomaly fires once until resolved.
↓
4. Alert fires
→ store.insertEvent (kind: 'anomaly') — persisted, visible on chart → RED runtime event → Node-RED notification bar → perf-monitor flow node output fires alarm msg

Heap growing at 62 MB/hr over last 30 min (baseline: 8 MB/hr). Possible memory leak.
⚠ Security patterns detected automatically:
  • Sustained CPU > 90% → possible crypto-mining
  • Monotonic heap growth → memory leak / injection
  • Event loop blocked > 500ms → DoS pattern
  • Traffic drop to 0 → possible crash / kill signal
